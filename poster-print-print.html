Diagnosing Agent Failures at Scale

Automating the Agent Failure Taxonomy with an LLM-as-a-judge pipeline
Huy Huynh, Aditya Kumar, Ava Doan
WUniversity of Washington
The Problem

Personal agents need diagnosis, not just accuracy scores.

Agents that act on a user's personal data face strict precision requirements — sending the wrong email is not a retry-able mistake.

Chugh et al. hand-annotated 500+ failure logs for the Agent Failure Taxonomy (AFT), flagging the bottleneck:

Annotating these traces by hand is laborious, challenging, and unfeasible to scale.

Our contribution: an LLM-as-a-judge pipeline that reproduces AFT at scale.

Findings

What the judge taught us about judging.

1
Preprocessing stabilizes the judge. Compiled VCC views raise top-1 agreement across runs from 0.91 → 1.00 and cut flipped categories ~35%.
2
Structural similarity beats lexical. Call-sequence overlap predicts judge accuracy better than raw text overlap.
3
Hierarchy helps where the taxonomy splits cleanly (Q1 > 0.80 F1) and hurts where labels overlap (Output State Mismatch ↔ Formatting Error).
4
Remaining errors look like taxonomy ambiguity, not judge failure — low-confidence misses cluster on traces that fit multiple AFT leaves.
Next Quarter

From a better judge to a scaled AFT study.

  • Generalize across the AFT grid — extend VCC adapters to AppWorld and MCP-Universe.
  • Cross-model evaluation — Sonnet 4.5, Kimi K2, Qwen3-32B for judge robustness.
